`loadAgent` reads a standalone agent description file (JSON or YAML), validates it against a strict schema, and returns a live `Agent` ready to call. For content already in memory, `loadAgentFromString` accepts a raw string directly.

```ts
import { loadAgent } from "@comma-agents/core";

const agent = await loadAgent("./agents/researcher.yaml");
const result = await agent.call("What is TypeScript?");
console.log(result.text);
```

## Agent Description File

A description file defines a single agent ‚Äî its name, model, optional system prompt, optional tools, and optional generation parameters. The schema is strict: unknown fields are rejected.

Model strings support two formats: explicit `"providerID/modelID"` (e.g. `"openai/gpt-4o"`) or bare model IDs (e.g. `"gpt-4o"`) where the provider is auto-resolved from the catalog. See [Model Resolution](/docs/core/model) for details.

### YAML

```yaml
name: researcher
model: openai/gpt-4o
systemPrompt: You are a research assistant.
tools:
  - read
  - grep
providerOptions:
  openai:
    reasoningEffort: high
modelOptions:
  temperature: 0.7
  maxOutputTokens: 4096
```

### JSON

```json
{
  "name": "researcher",
  "model": "openai/gpt-4o",
  "systemPrompt": "You are a research assistant.",
  "tools": ["read_file", "search_files"],
  "providerOptions": {
    "openai": { "reasoningEffort": "high" }
  },
  "modelOptions": {
    "temperature": 0.7,
    "maxOutputTokens": 4096
  }
}
```

### providerOptions

Provider-specific options forwarded verbatim to the AI SDK. The outer key is the provider ID (e.g. `"openai"`, `"anthropic"`) and the inner record contains the provider's specific options.

```yaml
providerOptions:
  anthropic:
    thinking: { type: enabled, budgetTokens: 8000 }
  openai:
    reasoningEffort: high
```

### modelOptions

Provider-agnostic generation parameters forwarded to every call. These map directly to the AI SDK's `streamText` options.

| Field | Type | Description |
|---|---|---|
| `temperature` | `number?` | Sampling temperature (0‚Äì2). Higher = more creative. |
| `topP` | `number?` | Nucleus sampling cumulative probability cutoff (0‚Äì1). |
| `topK` | `number?` | Top-K sampling ‚Äî limits the pool to K tokens. |
| `maxOutputTokens` | `number?` | Maximum output tokens per completion. |
| `maxRetries` | `number?` | Maximum retry attempts on transient failures. |
| `frequencyPenalty` | `number?` | Frequency penalty (‚àí2 to 2). Reduces repetition. |
| `presencePenalty` | `number?` | Presence penalty (‚àí2 to 2). Encourages new topics. |
| `seed` | `number?` | Deterministic seed for reproducible outputs. |

## loadAgentFromString

When the description content is already in memory (e.g., from a database, WebSocket, or test fixture), use `loadAgentFromString` directly:

```ts
import { loadAgentFromString } from "@comma-agents/core";

const yaml = `
name: writer
model: openai/gpt-4o
systemPrompt: You are a creative writer.
`;

const agent = await loadAgentFromString(yaml, "yaml");
const result = await agent.call("Write a haiku about TypeScript.");
```

The `format` parameter must be `"json"` or `"yaml"`. `loadAgent` auto-detects format from the file extension (`.json`, `.yaml`, `.yml`).

## System Prompt Templates

Instead of a static `systemPrompt`, you can define a `systemPromptTemplate` using [Liquid](https://liquidjs.com/) syntax with default variables:

```yaml
name: greeter
model: openai/gpt-4o
systemPromptTemplate:
  template: "You are a {{ role }} who speaks {{ language }}."
  variables:
    role: friendly assistant
    language: English
```

When both `systemPromptTemplate` and `systemPrompt` are present, the template takes precedence.

## Model and Tool Resolution

Model and tool names in the description file are plain strings (e.g., `"openai/gpt-4o"`, `"read"`). They are resolved at call time through the global registries ‚Äî `registerProvider()`, `registerModel()`, and `registerTool()`. Make sure the required providers and tools are registered before calling the agent.

## Error Handling

Both `loadAgent` and `loadAgentFromString` throw `StrategyValidationError` for:

- Unsupported file extensions (only `.json`, `.yaml`, `.yml`)
- Missing files
- Malformed JSON or YAML
- Schema validation failures (missing `name` or `model`, unknown fields, wrong types)

```ts
import { loadAgent, StrategyValidationError } from "@comma-agents/core";

try {
  const agent = await loadAgent("./agents/broken.yaml");
} catch (error) {
  if (error instanceof StrategyValidationError) {
    console.error("Invalid agent description:", error.message);
  }
}
```
