# Conversation Context

Conversation context tracks the history between a user and an agent. Each turn pairs a user message with the model's full response, including any tool calls and tool results. You can limit context with a sliding window by turn count, estimated token budget, or both.

## createConversationContext

Creates a `ConversationContext` instance. By default, context is unbounded (every turn is kept forever). Set `maxTurns` or `maxTokens` to enable automatic eviction of the oldest turns.

```ts
import { createConversationContext } from "@comma-agents/core";

// Unbounded — all turns are kept
const context = createConversationContext();

// Sliding window by turn count
const bounded = createConversationContext({ maxTurns: 20 });

// Sliding window by token budget
const tokened = createConversationContext({ maxTokens: 8000 });

// Both limits combined (whichever is stricter wins)
const combined = createConversationContext({ maxTurns: 20, maxTokens: 8000 });

// Explcit "none" strategy (same as the default with no limits)
const none = createConversationContext({ strategy: "none" });
```

## Using with an Agent

Append each completed turn to the context after the agent finishes responding. The context preserves native AI SDK message types so tool calls, reasoning, and multi-modal content survive across turns.

```ts
import { createConversationContext, createAgent } from "@comma-agents/core";

const context = createConversationContext({ maxTurns: 10 });
const agent = createAgent({ name: "assistant", model: "openai/gpt-4o" });

// Turn 1
const result1 = await agent.generate("What is TypeScript?");
context.append("What is TypeScript?", result1.response.messages);

// Turn 2
const result2 = await agent.generate(
  `Here is our conversation so far:\nRemember: the user asked about TypeScript.`,
  context.allMessages(),
);
context.append("Tell me about generics.", result2.response.messages);

// Inspect state
console.log(context.length);       // 2
console.log(context.isEmpty);      // false
console.log(context.estimateTokens()); // rough token count

// Serialize and restore
const snapshot = context.snapshot();
context.clear();
console.log(context.isEmpty);      // true
context.restore(snapshot);
console.log(context.length);       // 2
```

## Context Strategies

The default strategy depends on your configuration:

- If you set `maxTurns` or `maxTokens`, the strategy defaults to `"sliding-window"`.
- If you set neither, the strategy defaults to `"none"` (unbounded).

You can force an explicit strategy:

```ts
const ctx = createConversationContext({ strategy: "sliding-window" });
```

When `maxTurns` and `maxTokens` are both set, turns are evicted from the oldest until both constraints are satisfied. Token estimation uses a character-based heuristic (default: 0.25 tokens per character, roughly 4 characters per English token). Override `tokensPerChar` if your model tokenizes differently.
