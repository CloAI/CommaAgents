---
title: bash
description: Execute shell commands in a subprocess, capturing stdout and stderr.
---

The bash tool executes shell commands via `sh -c`, captures stdout and stderr, and returns the output. It supports configurable timeouts and working directories.

```ts
import { createAgent } from "@comma-agents/core";

const agent = createAgent({
  name: "runner",
  model: "openai/gpt-4o",
  tools: ["bash"],
});
```

## Parameters

The agent provides these parameters when invoking the tool:

| Parameter | Type | Description |
|---|---|---|
| `command` | `string` | The shell command to execute. |
| `timeout` | `number?` | Optional timeout in milliseconds. Defaults to 120,000 (2 minutes). |
| `workdir` | `string?` | Optional working directory for the command. |

## Behavior

- Commands run in a subprocess via `sh -c`.
- If the command exceeds the timeout, it receives `SIGTERM` followed by `SIGKILL` after 1 second.
- The tool propagates the abort signal from the agent context — aborting the agent also kills the subprocess.
- Output includes stdout, stderr (prefixed with `[stderr]`), and a non-zero exit code if applicable.

## Overriding Defaults

To customize the bash tool's defaults (e.g., timeout or working directory), define a replacement tool with `defineTool` and register it as `"bash"` to shadow the built-in:

```ts
import { defineTool, registerTool } from "@comma-agents/core";
import { z } from "zod";

const customBash = defineTool({
  description: "Custom bash with 60s timeout.",
  parameters: z.object({
    command: z.string(),
    timeout: z.number().optional(),
    workdir: z.string().optional(),
  }),
  execute: async ({ command, timeout, workdir }) => {
    // your custom implementation
    return { output: "..." };
  },
});

registerTool("bash", customBash);
```

A warning is logged when you shadow a built-in tool.
