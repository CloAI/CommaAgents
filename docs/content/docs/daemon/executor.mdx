# Executor

The executor module orchestrates strategy execution through a modular systems architecture. Each system handles a specific concern (input collection, permissions, streaming, persistence, etc.) and communicates through shared data stores and action registries.

## Architecture Overview

Systems communicate through:
- **SystemDataStore**: Shared state across systems (bridges, collectors, etc.)
- **RunActionRegistry**: Message routing between daemon and systems
- **Lifecycle Hooks**: `onRunPrepare`, `onStrategyLoaded`, `onBeforeExecute`, `onRunComplete`, `onRunError`, `onRunCleanup`

## Systems

### InputSystem

Manages user input collection and client responses.

<AutoTypeTable path="../packages/daemon/src/executor/systems/input/input.ts" name="InputSystem" />

**Responsibilities:**
- Stores `inputCollector` in systemData for strategy loading
- Registers `resolveInput` action for handling user responses
- Rejects pending input during run cleanup

### PermissionSystem

Manages permission requests and client decisions.

<AutoTypeTable path="../packages/daemon/src/executor/systems/permission/permission.ts" name="PermissionSystem" />

**Responsibilities:**
- Stores `permissionRequester` in systemData for sandbox callbacks
- Registers `resolvePermission` action for handling user decisions
- Rejects pending permission requests during run cleanup

### QuestionSystem

Manages question requests and client responses.

<AutoTypeTable path="../packages/daemon/src/executor/systems/question/question.ts" name="QuestionSystem" />

**Responsibilities:**
- Stores `questionRequester` in systemData for sandbox callbacks
- Registers `resolveQuestion` action for handling user responses
- Rejects pending question requests during run cleanup

### StreamingSystem

Manages agent and flow hooks for event broadcasting.

<AutoTypeTable path="../packages/daemon/src/executor/systems/streaming/streaming.ts" name="StreamingSystem" />

<AutoTypeTable path="../packages/daemon/src/executor/systems/streaming/streaming.ts" name="StreamingSystemOptions" />

**Responsibilities:**
- Builds and attaches agent hooks (beforeCall, onStreamEvent, afterCallResult)
- Builds and attaches flow hooks (beforeStep, afterStep)
- Broadcasts events to connected clients via EventSink
- Logs tool events for debugging

### PersistenceSystem

Manages run event persistence to disk.

<AutoTypeTable path="../packages/daemon/src/executor/systems/persistence/persistence.ts" name="PersistenceSystem" />

- Appends `run_started` event on strategy load

### SandboxSystem

Manages sandbox creation and policy enforcement.

<AutoTypeTable path="../packages/daemon/src/executor/systems/sandbox/sandbox.ts" name="SandboxSystem" />

**Responsibilities:**
- Creates sandbox with permission and question callbacks
- Registers `updatePolicy` action for runtime policy changes
- Enforces file system access policies

### SteeringSystem

Manages steering message injection.

<AutoTypeTable path="../packages/daemon/src/executor/systems/steering/steering.ts" name="SteeringSystem" />

**Responsibilities:**

### RunActionRegistry

Message routing registry for daemon-to-system communication.

<AutoTypeTable path="../packages/daemon/src/executor/systems/systems.types.ts" name="RunActionRegistry" />

**Registered Actions:**
- `resolveInput`: Routes user input to InputSystem
- `resolvePermission`: Routes permission decisions to PermissionSystem
- `resolveQuestion`: Routes question responses to QuestionSystem
- `updatePolicy`: Routes policy updates to SandboxSystem
- `steer`: Routes steering messages to SteeringSystem

## Lifecycle Hooks

Systems can hook into the execution lifecycle through these hooks:

<AutoTypeTable path="../packages/daemon/src/executor/systems/systems.types.ts" name="DaemonSystem" />

### Hook Execution Order

1. **onRunPrepare**: Called while preparing the run (InputSystem, PermissionSystem, QuestionSystem, SteeringSystem, SandboxSystem, SubLaunchSystem)
2. **onStrategyLoaded**: Called after strategy is loaded (StreamingSystem, PersistenceSystem, SandboxSystem)
3. **onBeforeExecute**: Called before flow execution (reserved for future use)
4. **onRunComplete**: Called on successful completion (PersistenceSystem)
5. **onRunError**: Called on error (PersistenceSystem)
6. **onRunCleanup**: Called when execution ends (InputSystem, PermissionSystem, QuestionSystem)
