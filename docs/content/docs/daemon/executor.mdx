# Executor

The executor module orchestrates strategy execution through a modular systems architecture. Each system handles a specific concern (input collection, permissions, streaming, persistence, etc.) and communicates through shared data stores and action registries.

## Architecture Overview

The executor uses a **systems-based architecture** where each system is responsible for a single concern:

```
┌─────────────────────────────────────────────────────────────┐
│                    StrategyExecutor                          │
│                                                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │ InputSystem  │  │PermissionSys │  │QuestionSystem│     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
│                                                             │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐     │
│  │SteeringSystem│  │StreamingSys  │  │PersistenceSys│     │
│  └──────────────┘  └──────────────┘  └──────────────┘     │
│                                                             │
│  ┌──────────────┐  ┌──────────────┐                        │
│  │ SandboxSystem│  │SubLaunchSys  │                        │
│  └──────────────┘  └──────────────┘                        │
└─────────────────────────────────────────────────────────────┘
```

Systems communicate through:
- **SystemDataStore**: Shared state across systems (bridges, collectors, etc.)
- **RunActionRegistry**: Message routing between daemon and systems
- **Lifecycle Hooks**: `onRunStart`, `onStrategyLoaded`, `onBeforeExecute`, `onRunComplete`, `onRunError`, `onRunCleanup`

### RunActionRegistry

Message routing registry for daemon-to-system communication.

<AutoTypeTable path="../packages/daemon/src/executor/systems/systems.types.ts" name="RunActionRegistry" />

**Registered Actions:**
- `resolveInput`: Routes user input to InputSystem
- `resolvePermission`: Routes permission decisions to PermissionSystem
- `resolveQuestion`: Routes question responses to QuestionSystem
- `updatePolicy`: Routes policy updates to SandboxSystem
- `steer`: Routes steering messages to SteeringSystem

## Lifecycle Hooks

Systems can hook into the execution lifecycle through these hooks:

<AutoTypeTable path="../packages/daemon/src/executor/systems/systems.types.ts" name="DaemonSystem" />

### Hook Execution Order

1. **onRunStart**: Called when execution begins (InputSystem, PermissionSystem, QuestionSystem, SteeringSystem, SandboxSystem, SubLaunchSystem)
2. **onStrategyLoaded**: Called after strategy is loaded (StreamingSystem, PersistenceSystem, SandboxSystem)
3. **onBeforeExecute**: Called before flow execution (reserved for future use)
4. **onRunComplete**: Called on successful completion (PersistenceSystem)
5. **onRunError**: Called on error (PersistenceSystem)
6. **onRunCleanup**: Called when execution ends (InputSystem, PermissionSystem, QuestionSystem)
