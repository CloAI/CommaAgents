# Hide

A responsive visibility component. Hides its children when the terminal width is below or above a threshold. Accepts named breakpoints or raw column counts.

```tsx
import { Hide } from "@comma-agents/tui";

// Hide on narrow terminals
<Hide below="md">
  <DetailedSidebar />
</Hide>

// Hide when wider than 128 columns
<Hide above={128}>
  <CompactView />
</Hide>
```

## Props

| Prop | Type | Description |
|---|---|---|
| `below` | `BreakpointName \| number` | Hide children when terminal width is below this threshold |
| `above` | `BreakpointName \| number` | Hide children when terminal width is at or above this threshold |
| `children` | `ReactNode` | Content to conditionally render |

## Breakpoints

Named breakpoints are resolved from the theme's `breakpoints` map. You can pass a breakpoint name (`"sm"`, `"md"`, `"lg"`, etc.) or a raw column number.

When `below` is set and the terminal has fewer columns than the threshold, children are not rendered. When `above` is set and the terminal has columns equal to or exceeding the threshold, children are not rendered. Both props can be combined.

## How It Works

`Hide` reads the current column count from the `useBreakpoint` hook and the breakpoint definitions from the theme. It returns `null` when the visibility condition is not met, otherwise renders children directly (no wrapper element).
